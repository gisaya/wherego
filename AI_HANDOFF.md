## 현재 제품 결정

- 질문은 총 7개다. 원천 3개, 검색 영향이 큰 일반 3개, 약한 재미 질문 1개를 한 세트 안에서 중복 없이 섞는다.
- 질문과 선택지의 단일 원본은 `jbg/apps/server/backend/app/resources/wherego/`다. 앱에는 로컬 질문 fallback이 없다.
- 사용자는 현재 위치 또는 직접 선택한 출발 지역을 사용한다.
- 후보 준비는 한국관광공사 KorService2와 DataLab을 사용하고 Gemini를 호출하지 않는다.
- 서버가 유효성, 거리, 하드 제약, 동일 장소 군집화, 방문자수 신호를 적용해 5~7개로 압축한다.
- Gemini 3.1 Flash-Lite는 압축 후보와 답변 메타데이터를 보고 최종 1개와 짧은 근거를 만든다. 모든 후보가 핵심 답변과 어긋날 때만 국내 장소를 직접 제안하고 관광공사 정확 일치 검색으로 재검증한다.
- Gemini 직접 제안이 관광공사에 없으면 `source=gemini`, `ktoVerified=false`로 명시한 자체 추천을 마지막 수단으로 반환한다. 관광공사 API 오류, 거리·지역 제약 위반, 반려동물 검증 필요 상황은 자체 추천으로 우회하지 않고 기존 후보를 사용한다.
- 실제 길찾기 API는 사용하지 않는다. 거리는 직선거리 기반 도로거리/시간 추정치이며 지도 열기는 네이버지도 검색 링크를 사용한다.
- 방문자수는 지역 단위 참고 신호다. 장소 단위 실시간 혼잡도로 표현하지 않는다.

## 현재 앱 흐름

1. `/` 또는 `/promotion` 진입
2. 잔여 횟수가 0이면 첫 화면에서 광고 충전 또는 10회 이용권 구매를 즉시 실행
3. 출발 기준 선택
4. 서버 질문 7개 수신
5. 카드 선택, 질문별 배너 광고
6. 마지막 답변 후 무료 후보 준비
7. 결과 전 짧은 전면광고 노출
8. 광고 `show`/`impression` 이후 Gemini 추천 시작
9. 필요 시 AI 로딩 화면
10. 결과 카드, 저장, 네이버지도, 홈 복귀

라우트 파일은 얇게 유지한다.

- `pages/index.tsx`: 일반 진입 `/`
- `pages/promotion.tsx`: 혜택 진입 `/promotion`
- `src/WheregoApp.tsx`: 공용 앱 UI와 상태 흐름
- `src/api/wheregoApi.ts`: 서버 API
- `src/promotion/resultPromotion.ts`: 프로모션 SDK와 로컬 1회 방어

- 일반 진입과 프로모션 진입을 분리하고 공용 UI를 `src/WheregoApp.tsx`로 이동해 페이지 초기화 순환을 제거했다.

- 이용권 구매 카드에 `이용권 횟수 사용 시 결과 전 전면광고 없음`과 미사용 횟수 무기한 보관을 핵심 혜택으로 표시한다.
- 첫 화면에서 전체 추천 잔여 횟수가 0이면 리워드 광고를 미리 준비하고 `광고 보고 1회 받기`, `10회 이용권 구매하기`를 바로 실행한다. 일반·프로모션 진입 모두 별도 화면 이동이 없다.
- 충전 화면은 시작 시점과 서버 예약 시점의 횟수가 달라졌거나 마지막 답변 뒤 서버가 횟수 부족을 반환한 예외 상황의 fallback으로 유지한다.
- 결과 전 광고 안내를 `짧은 광고`로 통일했다.
- 목적지 카테고리가 있는 일반 질문은 원천 의도 다음 검색 슬롯을 보장한다. 캠핑 타입 `28`, 문화시설 타입 `14`처럼 선택 메타데이터와 일치하는 후보가 있으면 Gemini와 규칙 fallback 모두 해당 타입에서 최종 선택한다.
- 문화공간 원천 선택지에서 전망대를 제거하고 박물관·미술관·전시관 중심으로 정리했다. AI 근거에 선택 장소가 뒷받침하지 않는 장소 유형이 나오면 제거한다.
- 최소 거리 하한이 있는 질문은 서버가 전국 검색으로 전환한다. `캠핑·피크닉 제외`에는 캠핑 검색 의도를 붙이지 않는다.
- Codex `qc` 자동화는 3시간마다 최근 결과를 검사하고, 최신 코드에서도 재현되는 좁은 서버·질문 메타데이터 결함만 테스트 후 보완한다.
- 최종 장소의 대표 이미지가 저장 카드에 부적합하면 `detailImage2`를 1회 조회하고 수정 가능한 `Type1` 이미지만 사용한다.
- Gemini가 기존 후보를 모두 부적합하다고 판단하면 장소명으로 관광공사를 한 번 재조회한다. 정확 일치가 없을 때만 출처가 표시된 AI 자체 추천과 일반 소개를 사용하며 운영시간·주차·입장료는 생성하지 않는다.
