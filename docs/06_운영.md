## 사용량 정책

- 기본 2회/일
- 리워드 광고 +1, 하루 최대 2회
- 친구 공유 +3, 하루 1회
- 유료 3회권, 공급가 450원·판매가 495원, 만료 없음
- KST 자정에 일일 항목만 초기화
- 후보 준비 예약, 성공 확정, 실패 환불, 중복 세션 멱등 처리

정책의 단일 원본은 JBG 서버 응답이다. 클라이언트 숫자는 서버가 로딩되기 전의 동일한 안내값일 뿐이다.

## 프로모션 운영

- 혜택 placement는 반드시 `/promotion`으로 등록한다.
- 일반 `/`에서는 혜택 문구와 프로모션 지급 API 호출이 없어야 한다.
- JBG는 `getAnonymousKey()` hash를 mTLS 검증하고 지급 key 발급과 실제 50원 지급을 수행한다.
- 서버 guard가 사용자 hash와 프로모션 코드 조합을 원자적으로 예약한다.
- 검증·지급 key 발급 전 실패만 안전하게 해제한다. 실제 지급 요청 결과가 불명확하면 중복 방지를 위해 예약을 유지한다.
- `/api/wherego/promotion/attempt`는 이전 출시 AIT 호환용으로 제거하지 않는다.

## 인앱결제 운영

- Apps in Toss 콘솔의 소모성 3회권 SKU와 상품 노출을 활성화한다.
- 클라이언트는 SDK 상품 응답의 실제 가격을 표시한다.
- Render는 로그인 secret과 mTLS 인증서를 사용해 주문을 검증한다.
- 유료 이용권 잔액이 있거나 마지막 유료 추천을 진행 중이면 배너광고가 노출되지 않는다.
- 주문 ID 기준 중복 지급을 막고 pending 주문 복원과 환불 회수를 수행한다.
- 인증서 원문과 private key는 Render secret에만 둔다.

## 자동 QC

자동 QC는 3시간마다 최근 3시간을 본다. 표본이 10건 미만이면 24시간을 함께 보고, 월요일 첫 실행에는 7일 편향 검사를 추가한다.

- 성공/실패와 후보 0건
- Gemini fallback
- 이미지·지도 누락
- 질문 7개와 구성
- 후보 5~7개
- 거리·지역 위반
- p95 지연
- 관광지·지역·검색 키워드 집중
- 답변과 추천/근거의 의미 불일치

원시 QC JSON, 사용자 해시, 세션 원문, DB 접속정보는 외부에 노출하지 않는다.
