## 현재 상태

- Git 저장소 위치: `C:\Users\ESOL\Documents\wherego`
- GitHub remote: `https://github.com/gisaya/wherego.git`
- Granite React Native 기반 Apps in Toss 앱 코드가 준비되어 있다.
- 앱 로고와 썸네일은 생성 완료.
- 약관/개인정보 처리방침 정적 페이지는 `뭐샀지` 프로젝트의 Vercel 정적 페이지 구조를 기준으로 추가 완료.
- 추천 API는 기존 `jbg` Render FastAPI 서버의 `POST /api/wherego/recommend`를 사용한다.

## 제품 방향

- 빠른 선택형 취향 테스트를 통해 국내 여행지를 추천한다.
- 복잡한 실시간 혼잡도/교통 데이터는 MVP에서 제외한다.
- 결과는 직관적인 카드 중심으로 보여주고 지도 앱 연결을 제공한다.
- 결과 화면 상단 헤더는 숨기고, 결과 카드 저장/지도 열기/처음으로 돌아가기 액션만 남긴다.

## 데이터 방향

- 관광지 기본 정보는 한국관광공사 국문 관광정보 서비스_GW API를 우선 검토한다.
- 공공데이터포털 개인 API 인증키와 한국관광공사 API 엔드포인트는 로컬 `.env.local`에 저장했다.
- 필요한 정보:
  - 관광지명
  - 주소
  - 좌표
  - 소개글
  - 대표 이미지
  - 지역/분류 코드
- AI 추천에는 사용자 답변과 관광지 후보의 특징만 전달한다.
- 서버는 선택지 메타데이터로 후보를 수집/압축하고, Gemini가 5개 이하 후보 중 최종 1개를 선택한다.

## 앱 구현 상태

- 첫 화면: 한국관광공사 관광정보와 방문자수 기반 추천 문구, 시작 버튼. 문구 블록은 화면 위쪽에 붙지 않게 중앙 쪽으로 내려 배치한다.
- 출발 기준: 현재 위치 권한 요청 또는 지역 선택 fallback.
- 질문: 원천 3개 + 일반 랜덤 5개, 2열 카드형 선택지.
- 광고: 질문 중 Apps in Toss 배너 광고, 결과 전 리워드 광고.
- 결과: 관광지 정보, AI 선택 근거, 카드 저장, 네이버지도 열기, 홈 복귀.
- 카드 저장: `react-native-svg` `toDataURL`로 PNG 결과 카드를 만든 뒤 Apps in Toss `saveBase64Data`로 저장. 미지원 Toss 버전은 텍스트 공유 fallback.

## 약한 부분

- PNG 카드 저장이 실제 Toss 기기와 SNS/갤러리에서 정상 동작하는지 검증해야 한다.
- Vercel GitHub 자동 배포 연결은 아직 되어 있지 않다.
- 관광공사 방문자수는 지역 단위 보조 신호이므로 장소 단위 혼잡도로 표현하면 안 된다.
- 이동 시간은 실제 경로 API가 아니라 추정값이다.
