# 구조와 흐름

최종 갱신: 2026-07-09 KST

## 사용자 흐름

```text
앱 진입
-> 여행 취향 질문
-> 답변 수집
-> 여행 성향 분석
-> 여행지 후보 매칭
-> 추천 결과 카드
-> 지도 앱 연결
```

## 추천 흐름

1. 사용자가 필수 질문 3개와 랜덤 질문 5개에 답한다.
2. 앱은 답변을 `tags`, `searchHints`, `constraints`로 구조화한다.
3. 서버는 Gemini Flash-Lite에 구조화 답변과 함수 목록을 넘겨 관광지 검색 계획을 만든다.
4. 서버는 한국관광공사 국문 관광정보 API로 후보를 찾고 상세/이미지/편의 정보를 보강한다.
5. 서버는 사용자 위치 또는 선택 지역 기준으로 이동 가능 후보만 남긴다.
6. 서버는 한국관광공사 지역별 방문자수 API로 지역 혼잡 신호를 붙인다.
7. AI는 여행 성향 문장, 한 줄 평, 추천 이유를 생성한다.
8. 앱은 추천 여행지 1~3곳과 지도 연결 버튼을 보여준다.

## 질문 구조

- 필수 질문 3개: 이동 스케일, 동행/제약, 목적지 의도.
- 랜덤 질문 5개: 서로 다른 태그 그룹에서 1개씩 선택.
- 랜덤 질문 그룹은 `crowd`, `mobility`, `weather`, `landscape`, `activity`, `time_mood`, `culture_style`, `food_link`, `accessibility`, `season`, `photo`, `healing_energy`, `route_style`를 사용한다.
- 질문은 성격 테스트가 아니라 관광지 추천 입력이다. 모든 선택지는 검색어, 거리/시간 제약, 편의조건 필터, 혼잡도 가중치 중 하나로 연결되어야 한다.

## 위치와 거리 필터

- 사용자가 위치 권한을 허용하면 현재 위치를 기준으로 한다.
- 위치 권한이 없으면 사용자가 선택한 출발 지역을 기준으로 한다.
- 테스트 스크립트 기본값은 `서울시청 테스트 위치`다.
- 현재 MVP 추정식은 직선거리 x 1.35를 도로거리로 보고, 평균 45km/h 기준 왕복 시간을 계산한다.
- 왕복/편도 시간 조건에는 15분 버퍼를 둔다.
- 실제 경로 시간은 TMAP/지도 경로 API가 필요하므로 MVP에서는 추정값으로만 사용한다.

## 외부 API

한국관광공사 국문 관광정보 서비스_GW API:

- `searchKeyword2`: 키워드, 지역코드, 콘텐츠 타입으로 관광지 후보 검색
- `detailCommon2`: 주소, 좌표, 개요 등 공통정보
- `detailIntro2`: 주차, 이용시간, 휴무일, 유모차 등 관광지 편의정보
- `detailImage2`: 결과 카드 이미지

한국관광공사 빅데이터 지역별 방문자수_GW API:

- `locgoRegnVisitrDDList`: 기초지자체별 일자 방문자수
- `metcoRegnVisitrDDList`: 광역지자체별 일자 방문자수
- 장소 주소에서 시/군/구를 추출해 최신 외지인+외국인 방문자수를 붙인다.
- 최신값을 같은 조회 기간의 중앙값과 비교해 `여유`, `보통`, `약간 붐빔`, `붐빔`으로 표시한다.

AI 서비스:

- Gemini Flash-Lite 사용 예정
- function calling으로 관광지 검색, 상세조회, 방문자수 조회를 호출
- 여행 성향 요약
- 추천 문구 생성
- 후보지와 사용자 취향의 매칭 이유 생성

## 보안 경계

- 외부 API 키는 클라이언트에 포함하지 않는다.
- 서버 또는 API route에서 한국관광공사/Gemini를 호출한다.
- 공공데이터포털 인증키는 `.env.local`의 `PUBLIC_DATA_PORTAL_SERVICE_KEY`에서 읽는다.
- AI에는 이름, 연락처 같은 직접 식별 정보를 보내지 않는다.
- 위치 정보는 사용자가 허용하거나 직접 선택한 지역 기준으로만 처리한다.
