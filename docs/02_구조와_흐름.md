# 구조와 흐름

최종 갱신: 2026-07-08 KST

## 사용자 흐름

```text
앱 진입
-> 여행 취향 질문
-> 답변 수집
-> 여행 성향 분석
-> 여행지 후보 매칭
-> 추천 결과 카드
-> 지도 앱 연결
```

## 추천 흐름

1. 사용자가 동행, 분위기, 이동 방식, 선호 활동 등을 선택한다.
2. 앱은 답변을 구조화된 값으로 모은다.
3. 서버는 관광지 후보 데이터와 답변을 함께 분석한다.
4. AI는 여행 성향 문장, 한 줄 평, 추천 이유를 생성한다.
5. 앱은 추천 여행지 1~3곳과 지도 연결 버튼을 보여준다.

## 외부 API

한국관광공사 국문 관광정보 서비스_GW API:

- 지역 기반 관광정보
- 위치 기반 관광정보
- 키워드 검색
- 공통정보
- 이미지정보

AI 서비스:

- 여행 성향 요약
- 추천 문구 생성
- 후보지와 사용자 취향의 매칭 이유 생성

## 보안 경계

- 외부 API 키는 클라이언트에 포함하지 않는다.
- 서버 또는 API route에서 한국관광공사/Gemini를 호출한다.
- AI에는 이름, 연락처 같은 직접 식별 정보를 보내지 않는다.
- 위치 정보는 사용자가 허용하거나 직접 선택한 지역 기준으로만 처리한다.
