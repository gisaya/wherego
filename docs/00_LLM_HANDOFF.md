## 현재 핵심 상태

- 일반 진입은 `/`, 혜택 진입은 `/promotion`이다.
- 질문은 원천 3개 + 핵심 일반 3개 + 재미 일반 1개, 총 7개다.
- 서버가 한국관광공사 KorService2/DataLab 후보를 준비하고 5~7개로 압축한다.
- Gemini 3.1 Flash-Lite는 압축 후보를 우선 비교하고 모두 부적합할 때만 장소를 직접 제안한다. 관광공사 정확 일치 재조회가 없으면 출처가 표시된 AI 자체 추천을 마지막 수단으로 반환한다.
- 압축 후보가 0개여도 Gemini `direct` 추천과 관광공사 재검증을 실행한다. 최종 검증 후보가 없으면 출발지를 유지한 새 질문 세트로 돌아가며 동일 조건 재시도를 반복하지 않는다.
- 관광공사 대표 이미지가 없으면 바다·자연·문화·야외활동 공통 이미지를 사용하며 화면과 저장 카드에 실제 사진이 아님을 표시한다.
- 기본 추천 2회/일, 리워드 광고 +1 최대 2회/일, 공유 +3 하루 1회, 구매 10회권을 사용한다.
- 전체 잔여 횟수가 0이면 일반·프로모션 첫 화면에서 리워드 광고와 10회 이용권 구매를 화면 이동 없이 바로 실행한다.
- 첫 화면은 횟수·로그인·결제 복원·상품 조회가 끝난 뒤 한 번에 표시한다. 질문 배너는 2지선다·4지선다 모두 하단 안전영역 위에 고정한다.
- 성공 결과 2회 이상 사용자에게 앱 리뷰 요청 CTA를 한 번만 표시한다.
- 결과 전 전면광고는 무료·보상 횟수에 노출하고 유료 횟수에는 생략한다.
- `/promotion` 결과에서 `getAnonymousKey()` hash를 JBG가 mTLS로 검증한 뒤 운영 코드로 토스 포인트 50원을 1인 1회 서버 지급한다.
- 프로모션 서버 중복 방지 예약과 hash 검증이 실패하면 지급하지 않는 fail-closed 정책이다. 이전 AIT용 `/promotion/attempt`는 유지한다.
- 최소 거리 하한은 전국 검색으로 전환하고, 명시 지역은 질문 메타데이터 기반 하드 제약으로 적용한다.
- 가까운 이동 범위는 동일 행정도시 하드 제약 대신 생활권 45분, 인접 도시 90분 기준을 사용한다.
- 캠핑·문화시설처럼 관광 타입이 명시된 선택은 해당 타입 후보가 있을 때 Gemini와 규칙 fallback의 최종 허용 범위를 제한한다.
- 추천 결과 QC는 3시간마다 실행하며 표본이 적으면 24시간을 함께 본다.

## 수정 원칙

- 질문 카피와 메타데이터는 앱이 아니라 서버 JSON에서 수정한다.
- API 키, 인증서, 사용자 원문, 익명 사용자 식별값, `.env*`를 문서나 Git에 넣지 않는다.
- `.ait`, `.vercel/output`, `node_modules`, 빌드 캐시를 커밋하지 않는다.
- 사용자가 `저장`이라고 하면 `SAVE_PROTOCOL.md`에 따라 문서 갱신, 최소 검증, 커밋, 푸시까지 수행한다.
- 앱 코드 또는 네이티브 SDK 설정이 바뀐 저장에서만 AIT를 다시 만든다.
