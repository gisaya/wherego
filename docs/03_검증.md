# 검증

최종 갱신: 2026-07-16 KST

명령 원문은 루트 `RUNBOOK.md`를 따른다. 이 문서는 변경 유형별 합격 기준만 정의한다.

## 모든 앱 변경

- Jest 단위 테스트 통과
- TypeScript strict 검사 통과
- `git diff --check` 통과
- 비밀값, `TEST_` 코드, 생성물 미추적 확인
- 일반 `/`과 혜택 `/promotion`의 조건 분기 검토

## UI 변경

- Android 실기기에서 2지선다와 4지선다 카드 선택 가능
- 긴 질문·선택지·번호·작은 문구가 잘리지 않음
- 배너가 버튼이나 카드와 겹치지 않음
- 뒤로가기 종료 확인창이 모든 단계에서 동작
- 화면 저장 PNG가 1080x1350 비율이고 이미지·위치·AI 근거가 잘리지 않음

## 질문 또는 추천 변경

- 서버 Wherego 테스트 전체 통과
- 질문은 정확히 7개: 원천 3 + 핵심 일반 3 + 재미 1
- 한 세트 내부 질문 ID와 의미 테마 중복 0
- 모든 선택지가 검색 가능한 메타데이터를 가짐
- 선택된 혼잡·이동·접근성 메타데이터와 거리/지역/동행 하드 제약이 후보 압축 전에 적용됨
- 후보는 정상 흐름에서 5~7개, 최종 추천은 1개
- 명시 지역과 최소·최대 거리 위반 0

질문 JSON만 변경했으면 서버 배포와 API 확인으로 충분하며 AIT를 만들지 않는다.

## 광고 변경

- 출시 소스와 AIT에 라이브 ID만 존재
- 질문 배너, 결과 전 전면광고, 횟수 충전 리워드 광고의 역할이 섞이지 않음
- 리워드 횟수는 `userEarnedReward` 이후에만 지급
- 무료/보상 크레딧은 결과 전 광고, 유료 크레딧은 생략
- Gemini는 결과 전 광고가 표시되기 시작한 뒤 한 번만 호출

## 프로모션 변경

- `src/promotion/resultPromotion.test.ts`의 guard/SDK 경계 테스트 통과
- 일반 진입에서 프로모션 SDK 호출 0
- 혜택 진입 최초 성공 결과에서만 50원 지급
- 서버 guard 예약 실패 시 SDK 호출 0
- 재렌더, 재진입, 재설치에 중복 지급 0
- 불명확한 SDK rejection은 자동 재시도하지 않음
- 운영 AIT에 테스트 코드 없음

## 인앱결제 변경

- SDK 상품명·설명·이미지·가격 표시
- 구매 시점에만 로그인
- 성공 주문 +10 정확히 1회
- pending 복원, 취소 미지급, 중복 주문 미지급, 환불 회수
- 앱 재실행과 재로그인 후 유료 잔여 유지

## 서버 배포 후

- `/api/health`가 새 JBG commit과 Postgres 정상 상태를 반환
- `/api/wherego/questions`가 7문항과 올바른 구성 메타데이터를 반환
- `/api/wherego/usage`가 기본 2, 광고 한도 2를 반환
- IAP products가 운영 SKU를 반환
- 추천 smoke가 이미지·지도와 최종 1개를 반환

## QC 기준

6시간마다 최근 6시간을 본다. 표본이 10건 미만이면 24시간을 함께 보고, 월요일 첫 실행에는 168시간을 추가한다.

- 성공/실패율과 후보 0건
- Gemini fallback
- 이미지·지도 누락
- 7문항 및 원천3+일반4 구조
- 후보 압축 범위
- 거리·지역 제약 위반
- p95 전체/KTO/Gemini 지연
- 특정 관광지·지역·검색 키워드 집중도
- 답변과 최종 장소·AI 근거의 의미 일치

주간 표본 10건 미만에서는 편향을 단정하지 않는다.

최종 검증 결과와 현재 운영 커밋은 `AI_HANDOFF.md`에만 기록하고 이 문서에는 반복해서 복사하지 않는다.
